Team Recommendation — Overview (names only)

Scannable summary of the four-playthrough plan. No detailed builds here — see the per-route files for class paths, training, loadouts, recruitment, and pairings. Rules: team-rules.md · Per-route detail: routes/ Tags: 🔒 route-exclusive · ⚠️ tutor/special-care · 💃 dancer · ⭐ top carry · 🧩 DLC-optional (non-DLC alt given) · (core) / (flex) roster role

R7 — bigger rosters: every route trains 12–13 units (largest maps deploy up to 12, plus up to 3 adjutants). Each unit is marked core or flex/bench. With rosters this large some reuse across routes is unavoidable — a reused unit gets a different class (R3, "reuse-variant"). We still feature as many distinct units as possible (R1), but filling the roster (R7) takes priority over zero-overlap.

R5 — romance: give each unit their BEST pairing by default and maximize the paired-ending count on every route. Repeating a best pairing across routes is ALLOWED and expected when a unit recurs (a repeated best pairing beats a forced weak one). Sub-optimal "diversity" pairings are allowed ONLY on the low-difficulty routes — Azure Moon (Normal) first, then Crimson Flower (Hard). The two Maddening routes (Silver Snow, Verdant Wind) always use best pairings and never bench a unit just to avoid a repeat. A unit is left unpaired only if it shares no real support chain with an available partner; uncertain chains are flagged ⚠️/verify; none invented.
